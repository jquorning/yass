Types of tags

Here are two types of tags:

Global tags

They are defined for whole site in site.cfg file. This file have standard config file syntax: [Key] = [Value]. For example:

Name = Yet Another Static Site (Generator) Documentation
  • Each pair must be in one line. Either Key nor Value can't contains new line.
  • You can add as many as you want tags to project.

Local (page) tags

They are defined directly in pages content files (markdown files with .md extension). They have syntax: -- [Key]: [Value]. For example:

-- layout: default
  • Same as global tags, each pair must be in one line. Either Key nor Value can't contains new line.
  • Tag layout is only mandatory tag. Each markdown file (which later will be converted to HTML) must have this tag. This tag must have as Value name of existing file in layouts directory (without .html file extension).
  • Same as global tags, you can add as many as you want tags to project.
  • Tags can be added in any place of file (at beginning, between text, at end, etc) just they must start with new line. Program can't detect tags inside text.
  • Whole text from markdown file is stored in tag Content.

Both type of tags can have string, numeric or boolean values. For string and numeric value just enter value. String value example:

-- title: title of page

Numeric value example:

-- paragraphnumber: 12

For boolean values, enter as value true or false (both case-insensitive). Example:

-- havetitle: true

Using tags inside templates files

To use any tags inside template file you must surround selected tag with {% and %}. For example, to get value of tag Content add to template file:

{%Content%}

Note: All tags names are case-sensitive.

For complete documentation on how to use tags and what is available, please refer to Ada Web Server Templates documentation. Especially:

  • Tags: part about Filters, Attributes and Predefined Tags.
  • Template statements: whole section.
  • Macros: whole section.

Note: In YASS you will use {% to start a tag (instead of @_) and %} to end a tag (instead of _@).
